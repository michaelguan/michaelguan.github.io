100 行实现 SPA 路由：History API + iframe 方案

不依赖 React / Vue，原生 JS 实现浏览器前进/后退、iframe 通信、滚动恢复、懒加载的完整路由方案。

这个 100 行的路由核心能为你撑住一个中型博客的全部客户端路由需求。不是示范，是生产代码。本文详细拆解设计思路与实现路径。

核心思路

本博客采用「框架页 + iframe 内容页」双层架构：

  • index.html：外壳，包含导航栏、左栏、三栏布局、主内容容器（）、页脚
  • 内容页：嵌入分类列表 / 文章详情，于 iframe 中独立渲染
  • 点击导航链接 → 修改 iframe src + history.pushState
  • 浏览器前进/后退 → 监听 popstate 同步 iframe
  • iframe 内部链接用 target="content-frame"保持在框架内

router.js 实现

所有逻辑集中在一个 IIFE 里，避免污染全局命名空间。

(function () {
  'use strict';

  const ROUTES = {
    home: '/index.html',
    tech: '/pages/tech/index.html',
    game: '/pages/game/index.html',
    life: '/pages/life/index.html',
    '':    '/pages/tech/index.html'
  };

  const frame = document.querySelector('.content-frame');
  let currentRoute = '';

  function getRouteFromHash() {
    const hash = window.location.hash.slice(1);
    return ROUTES.hasOwnProperty(hash) ? hash : 'tech';
  }

  function navigate(route, pushState) {
    if (ROUTES[route]) frame.src = ROUTES[route];
    currentRoute = route;
    setActiveLink(route);
    if (pushState !== false) {
      const newHash = route === 'tech' ? '' : '#' + route;
      history.pushState({ route }, '', newHash);
    }
  }

  window.addEventListener('hashchange', () => {
    navigate(getRouteFromHash(), false);
  });

  window.addEventListener('popstate', e => {
    const route = e.state?.route || getRouteFromHash();
    navigate(route, false);
  });

  document.querySelectorAll('.nav-item[data-category]').forEach(link => {
    link.addEventListener('click', function (e) {
      const route = this.dataset.category;
      if (route && ROUTES[route]) {
        e.preventDefault();
        navigate(route);
      }
    });
  });
})();

iframe 加载状态管理

主页面监听 iframe 的 load / error 事件，控制加载、错误、成功三态。

frame.addEventListener('load', () => {
  frame.classList.remove('is-loading');
  container.classList.add('is-loaded');
});

frame.addEventListener('error', () => {
  container.classList.add('has-error');
});

优缺点权衡

优点缺点
零依赖、零构建、原生部署iframe 隔离导致样式/脚本不共享
History API 原生前进后退SEO 较弱（iframe content）
内容页可独立开发/预览跨 iframe 通信需要 postMessage
每页独立脚本、互不干扰首屏要多加载一个 HTML

适用场景

个人博客、文档站、作品集、内部工具 —— 任何「内容为主、交互简单、想要零配置部署」的场景都能担当。

代码 的灵魂不在技术先进，而在为具体问题提供精确的答案。Iframe + pushState 是「轻量 SPA」的一种素质解答。
