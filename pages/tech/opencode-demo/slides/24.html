/plan 模式详解

Plan 是 OpenCode 内置的受限主代理——切换它（Tab 或命令），代理进入"只读思考"阶段，把方案给你看清楚后再动手。

让 OpenCode 先当"思考伙伴"，而不是"执行机器"。

默认权限（全部 ask）

  • file edits：所有写入/补丁/编辑都需确认
  • bash：所有 bash 命令都需确认
  • 仅允许在 .opencode/plans/*.md 写计划

适合场景

  • 架构决策、跨系统重构
  • 安全审查、性能分析
  • 读代码、给建议但不改文件
# 在 plan 模式下提问
「分析我们的鉴权体系（src/auth/），给出 OAuth2 登录的落地方案。不要写任何代码，只要方案。」
