Hello World — 博客上线

新博客框架搭建完成：纯静态 HTML / CSS / JS + iframe 路由。记录从零开始的技术选型与实现细节。

signature: 3-column layout stack: Vanilla JS

新的博客已经搭建完成。整个站点采用纯静态 HTML / CSS / JS，没有构建步骤、没有依赖、没有框架。这种「最简单」的选择，反而最适合作者气质——我把这种坚持叫做反向最小化（Reverse Minimal）。

为什么重写？

原有的 michaelguan.github.io 是一个「诗词大冒险」小游戏加杂乱的个人页面。我想要一个真正属于自己的博客，记录技术思考、游戏感想、生活碎片。要求：

  • 零依赖、零构建、零配置
  • GitHub Pages 原生部署
  • 现代化视觉，支持暗色模式
  • 分类导航，三栏阅读体验

技术栈

这次重写几乎不引入任何外部依赖。所有的决策都基于「老派工程师」的偏好：能原生解决就原生解决。

层级 选择 理由
HTML 语义化标签 SEO、无障碍、可维护
CSS CSS 变量 + 模块文件 主题系统、无运行时、原生
JS Vanilla ES6+ (router.js) History API、iframe 通信， < 3KB
部署 GitHub Pages 免费、自动 HTTPS、自定义域名

核心实现：iframe 与 SPA 的折中

采用 index.html 作为框架页 + 内容 iframe。点击导航切换 iframe src + history.pushState。浏览器前进/后退通过 popstate 同步。

CAVEAT — Iframe 通讯的限制

每篇文章页是独立 http 请求。SEO 上无法索引（搜索引擎有时跳过 iframe content），但对于「个人展示」而言是可接受折中。如果未来流量成为目标，可以引入 11ty、VitePress 等静态生成工具。

目录结构

三栏布局：左栏分类书签、右栏 about + tags，中间内容容器（iframe）。

michaelguan.github.io/
├── index.html              # 框架页
├── libs/
│   ├── css/                # theme, layout, leftnav, sidebar, main, iframe, article, category
│   └── js/                 # layout, router
├── pages/
│   ├── tech/               # 技术分类
│   ├── game/               # 游戏分类
│   └── life/               # 生活分类
└── deploy.sh
