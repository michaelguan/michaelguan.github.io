GL Reconciler 工作流

1

摄入 GL 与子账数据

从 Box/Egnyte 获取 GL 总账与子账 Excel 数据，解析科目结构、交易流水与余额信息

data ingestion
2

Critic 子代理：扫描识别断差

比对总账与子账余额，识别所有不平衡账户，计算断差金额，按严重性分类（<1k / 1k-10k / >10k）

/gl-recon
3

Reader 子代理：追踪交易流水

对每个断差账户逐笔追踪交易流水，定位 Possible 断差交易（未匹配/重复/金额差异）

break-trace
4

Resolver 子代理：根因分析

分析断差模式，判断根因类型（Timing difference / Entry error / Missing transaction / System bug）

root cause analysis
5

分级路由处理

低金额断差自动 APPROVE 或 REJECT；中高金额 FLAGGED 路由人工复核；重大风险即时上报

tiered routing
6

输出对账报告待审批

生成完整对账报告，包含断差明细、根因分析、修复建议，等待财务负责人签字

report generation

Guardrails（安全机制）

>$10k 必须人工复核
根因须有证据链支撑
所有操作需记录审计轨迹
