多 Agent 接力工作流（handoff）

工作原理

Agent A {"type":"handoff_request",
"target_agent":"model-builder",
"payload":{"event":"..."}} Agent B
  1. Agent A 输出流中生成 handoff_request JSON blob
  2. orchestrate.py 监听 message_delta，正则捕获 blob
  3. 硬编码白名单校验（仅 10 个已部署 Agent slug）
  4. JSON Schema 校验 → steer() 引导到 Agent B
# orchestrate.py
ALLOWED_TARGETS = { "pitch-agent", "model-builder", ... }
# 校验通过后 steer
client.beta.agents.sessions.steer(
    agent_id=target_id,
    input=handoff["payload"]["event"])
