业务落地矩阵（一）— 智能体 × 银行岗位

智能体 银行岗位 业务流程 决策权限
pitch-agent
 投行部 · 高级分析师 可比公司 → 先例交易 → DCF/LBO 建模 → PitchBook 生成 草拟权 · 需 MD 审批
model-builder
 投行部 · 金融建模岗 DCF/LBO/三表模型构建 → 勾稽复核 → 敏感性分析 草拟权 · 需 VP 审批
valuation-reviewer
 投行/基金会计主管 季度投资组合估值审查 → LP 报告生成 → 瀑布分配 建议权 · 需 IR/CCO 签字
earnings-reviewer
 研究部 · 股票研究员 财报电话会读取 → 财务模型更新 → 研究笔记撰写 草拟权 · 需首席分析师
market-researcher
 研究部 · 行业研究员 行业概览 → 竞争格局 → 可比分析 → 创意筛选 → 研报 草拟权 · 需分析师
meeting-prep-agent
 客户经理 · 私行顾问 客户画像提取 → 持仓分析 → 市场背景 → 简报包生成 完全授权 · 仅供顾问参考
gl-reconciler
 运营部 · 总账会计 总账与子账逐日对账 → 差异追踪 → 异常报告 建议权 · 需主管签字
statement-auditor
 运营部 · 对账单审核岗 LP 对账单与 NAV 逐笔勾稽 → 差异标记 → 签字确认表 建议权 · 需人工签字
kyc-screener
 合规部 · 开户审核岗 开户文件解析 → 制裁/PEP 筛查 → 风险评级 → 上报包 建议权 · 需合规官
month-end-closer
 财务部 · 会计主管 应计项目计提 → 滚进计算 → 差异分析 → 结账包生成 草拟权 · 需财务总监
投行部 研究部 客户经理 运营部 合规部 财务部

核心发现：10 个智能体覆盖银行 5 个主要部门的核心业务流程；所有 Agent 输出均为草稿或建议，最终决策与分发均在人工控制下 —— 满足金融机构"人机协同"的监管要求。
